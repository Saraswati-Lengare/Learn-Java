  • 1. What are keywords in Java?
    Answer

    Keywords in Java are reserved words that have a predefined meaning in the language. They cannot be used as identifiers (like variable names, class names, etc.).

    Examples: class, public, static, void, etc.

  • 2. How many keywords are there in Java?
    Answer

    There are 50+ reserved keywords in Java (exact number depends on Java version). Common ones include if, else, for, while, switch, etc.

  • 3. Can keywords be written in uppercase in Java?
    Answer

    Java keywords are case-sensitive and must always be written in lowercase. For example, class is valid but Class is not.

  • 4. What is the difference between 'final', 'finally', and 'finalize' in Java?
    Answer
    Keyword Description
    final Used to declare constants, prevent inheritance, and stop method overriding.
    finally A block used in exception handling to execute important code (always runs).
    finalize A method used by the Garbage Collector before destroying an object.
  • 5. What is the difference between 'this' and 'super' keywords?
    Answer

    this refers to the current object of the class, while super refers to the immediate parent class object.

    class Parent {
    void show() { System.out.println("Parent"); }
}

class Child extends Parent {
    void show() {
        super.show(); // calls Parent method
        System.out.println("Child");
    }
}

public class Main {
    public static void main(String[] args) {
        Child c = new Child();
        c.show();
    }
}

    Output:

    Parent
Child

    Naming conventions make code readable, maintainable, and standardized across teams. They also help in avoiding confusion while debugging or collaborating.

    Variables should follow the camelCase convention (first word starts lowercase, subsequent words start uppercase).

    Examples: age, studentName, totalMarks

    Class names should follow PascalCase (each word starts with an uppercase letter).

    Examples: Student, EmployeeDetails, BankAccount

  • 5. What is the naming convention for methods in Java?
    Answer

    Method names should follow camelCase and generally represent an action.

    Examples: calculateSum(), getStudentName(), displayResult()

  • 6. What is the convention for constants in Java?
    Answer

    Constants should be written in UPPERCASE letters, with words separated by underscores.

    Examples: MAX_VALUE, PI, TAX_RATE

    Answer

    Package names should be written in lowercase to avoid conflicts.

    Examples: java.util, com.companyname.project

    Answer

    Interfaces should follow PascalCase and often use adjectives to describe capability.

    Examples: Runnable, Readable, Serializable

    Answer

    Array names should also follow camelCase, and usually represent plural nouns.

    Examples: students, marksList, employeeSalaries

    Answer

    Code will still compile and run, but it will be hard to read, maintain, and understand for other developers.

    Literals

    EASY QUESTIONS

  • 1. What are literals in Java?
    Answer

    Literals in Java are constant values that are directly assigned to variables. They represent fixed values in the program.

    Examples: 10, 'A', true, "Hello"

  • 2. What are the types of literals in Java?
    Answer

    Java provides several types of literals:

    • Integer literals (e.g., 10, -25)
    • Floating-point literals (e.g., 3.14, 2.5f)
    • Character literals (e.g., 'A', '9')
    • String literals (e.g., "Java", "Hello")
    • Boolean literals (true, false)
    • Null literal (null)
  • 3. What is the difference between integer and floating-point literals?
    Answer

    Integer literals represent whole numbers (e.g., 100), while floating-point literals represent decimal values (e.g., 10.5).

    • MEDIUM QUESTIONS

  • 4. What are character literals in Java?
    Answer

    Character literals represent a single character enclosed in single quotes.

    Examples: 'A', 'z', '9'

  • 5. What are string literals in Java?
    Answer

    String literals represent a sequence of characters enclosed in double quotes.

    Examples: "Hello", "Java Programming"

    Answer

    • Decimal (base 10) → 100
    • Octal (base 8, prefix 0) → 012
    • Hexadecimal (base 16, prefix 0x or 0X) → 0x1A
    • Binary (base 2, prefix 0b or 0B) → 0b1010

    Answer

    The null literal represents the absence of any object reference. It can only be assigned to reference types (not primitives).

  • 9. Can we use underscores in numeric literals?
    Answer

    Yes, Java allows underscores in numeric literals (since Java 7) to improve readability.

    Example: 1_000_000 (one million).

    Java data types are divided into two categories:

    • Primitive data types (e.g., int, float, char, boolean)
    • Non-primitive (reference) data types (e.g., String, Arrays, Objects)

    Local Variables

    EASY QUESTIONS

  • 1. What is a local variable in Java?
    Answer

    A local variable is a variable declared inside a method, constructor, or block and can only be accessed within that scope.

    
class Test {
    void display() {
        int x = 10; // local variable
        System.out.println(x);
    }
}
  • 2. What is the lifetime of a local variable?
    Answer

    A local variable exists only while the method or block is executing. After that, it is destroyed.

  • 3. Do local variables have default values?
    Answer

    No, local variables do not have default values. They must be explicitly initialized before use.

    
void test() {
    int num;
    // System.out.println(num); // ❌ Error: num not initialized
}

    • MEDIUM QUESTIONS

  • 4. Can local variables be accessed outside the method?
    Answer

    No, local variables are not accessible outside the method, constructor, or block in which they are declared.

    
class Test {
    void method() {
        int a = 5;
        System.out.println(a);
    }
}
Test t = new Test();
// System.out.println(t.a); // ❌ Error: a not accessible
  • 5. Can local variables be static?
    Answer

    No, local variables cannot be declared as static, because they belong to the method instance, not the class.

  • 6. Can a local variable shadow an instance variable?
    Answer

    Yes, a local variable with the same name as an instance variable will hide the instance variable within that method.

    
class Test {
    int x = 20;
    void show() {
        int x = 10; // shadows instance variable
        System.out.println(x); // prints 10
    }
}

    • HARD QUESTIONS

  • 7. Can local variables be final?
    Answer

    Yes, local variables can be declared as final. Once initialized, their values cannot be changed.

    
void method() {
    final int num = 50;
    // num = 60; // ❌ Error: cannot reassign final variable
}
  • 8. What is a local variable in a for loop?
    Answer

    A loop variable declared inside a for loop is also a local variable and exists only within the loop.

    
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}
// System.out.println(i); // ❌ Error: i not accessible
  • 9. Why must local variables be initialized before use?
    Answer

    Because Java compiler does not automatically assign default values to local variables, they must be explicitly initialized before use.

    Global Variables

    EASY QUESTIONS

  • 1. What is a global variable in Java?
    Answer

    A global variable in Java is typically a static variable declared inside a class but outside any method, constructor, or block. It can be accessed throughout the class and from other classes using the class name.

    
class Test {
    static int count = 100; // global variable
}
  • 2. What is the default value of a global variable?
    Answer

    Global variables (static variables) are automatically assigned default values (e.g., int = 0, boolean = false, object = null).

  • 3. Where are global variables stored in memory?
    Answer

    Global variables are stored in the method area (class area) of memory and are shared across all objects of the class.

    • MEDIUM QUESTIONS

  • 4. How do we access a global variable from another class?
    Answer

    Global variables can be accessed using the class name directly without creating an object.

    
class Test {
    static int num = 50; // global variable
}
class Main {
    public static void main(String[] args) {
        System.out.println(Test.num); // Accessing global variable
    }
}
  • 5. Can global variables be private?
    Answer

    Yes, a global variable can be declared private, but then it can only be accessed within the same class. To access it outside, we use getter and setter methods.

  • 6. Can global variables be changed?
    Answer

    Yes, unless they are declared as final, global variables can be modified and the change is reflected across all objects of the class.

    • HARD QUESTIONS

  • 7. Can global variables be final?
    Answer

    Yes, if a global variable is declared final, its value cannot be changed once initialized.

    
class Test {
    static final int MAX = 100; // constant global variable
}
  • 8. How do global variables differ from local variables?
    Answer

    Global variables (static) have class-wide scope and default values, while local variables are method-scoped and must be explicitly initialized before use.

  • 9. Can global variables cause memory issues?
    Answer

    Yes, global variables remain in memory as long as the class is loaded, so excessive use can cause high memory consumption and make debugging harder.

  • 10. Can we use global variables in multithreaded programs?
    Answer

    Yes, but global variables can cause data inconsistency in multithreaded environments if multiple threads modify them simultaneously. To handle this, we use synchronization or volatile keyword.

    
class Counter {
    static int count = 0;
    synchronized void increment() {
        count++;
    }
}

    Arithmetic Operators

    EASY QUESTIONS

  • 1. What are arithmetic operators in Java?
    Answer

    Arithmetic operators in Java are used to perform basic mathematical operations like addition, subtraction, multiplication, division, and modulus.

  • 2. List the arithmetic operators in Java.
    Answer

    The arithmetic operators in Java are:

    • + → Addition
    • - → Subtraction
    • * → Multiplication
    • / → Division
    • % → Modulus (remainder)
  • 3. What is the result of division between two integers?
    Answer

    If both operands are integers, the result will also be an integer (decimal part is truncated).

    
System.out.println(7 / 2); // Output: 3

    • MEDIUM QUESTIONS

  • 4. What is the difference between division (/) and modulus (%)?
    Answer

    Division (/) gives the quotient, while modulus (%) gives the remainder.

    
System.out.println(10 / 3); // Output: 3
System.out.println(10 % 3); // Output: 1
  • 5. Can arithmetic operators be used with characters?
    Answer

    Yes, since characters are stored as Unicode values, arithmetic operations can be performed on them.

    
char c = 'A';
System.out.println(c + 1); // Output: 66 (since 'A' = 65)
  • 6. What happens when division is performed by zero?
    Answer

    If the divisor is zero:

    • For integers → ArithmeticException
    • For floating-point → Infinity or NaN

    
System.out.println(10 / 0);   // Exception
System.out.println(10.0 / 0); // Infinity

    • HARD QUESTIONS

  • 7. How do arithmetic operators behave with mixed data types?
    Answer

    In Java, smaller data types are promoted to larger ones during operations. For example, int + double → double.

    
int a = 5;
double b = 2.5;
System.out.println(a + b); // Output: 7.5
  • 8. What is the effect of overflow in arithmetic operations?
